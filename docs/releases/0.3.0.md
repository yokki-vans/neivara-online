# 0.3.0 — characters and Dawnmere vertical slice

Status: production-oriented vertical slice and public technical preview.

This release is **not** a claim that Neivara is a complete production MMORPG or an AAA game. It proves a coherent browser gameplay loop, original 3D content pipeline and hardened server foundation; content breadth, scale validation and final manual art polish remain future gates.

## Characters

- five selectable races: humans, light elves, dark elves, dwarves and orcs;
- male and female presentation for the Warrior and Mage classes;
- 20 skinned GLB character variants (`5 × 2 × 2`) with embedded original textures;
- interactive Babylon 3D preview updates before character creation;
- canonical `race`, `gender` and `classId` are guaranteed at the API/runtime boundary while PostgreSQL temporarily accepts both 0.2 aliases and canonical IDs;
- runtime idle/run/attack/cast/hit/death animation support through the shared model library.

The 20 variants are a complete functional matrix, not 20 hand-sculpted AAA hero productions. Manual retopology, facial/hair customization, LOD0–LOD2, KTX2/Basis, Meshopt and final motion polish remain art gates.

## Dawnmere Crossing

- the starter zone is now `dawnmere_crossing`, displayed as «Переправа Донмер»;
- the scene includes custom terrain, river crossing, paths, settlement architecture, ruins, cave approaches, vegetation and the PvP circle;
- six original creature archetypes populate separate spawn pockets: thorn prowler, moss mauler, cave shrieker, ruin sentinel, bramble boar and ember drake;
- creature names, level, health, elite state, AI combat stats, respawn, XP, gold and quest eligibility are data-driven;
- `first_echoes` keeps its durable quest ID and advances from several appropriate normal starter creatures, not elite guardians.

## Combat presentation

- original code-native SVG skill icon catalog for current and future Warrior/Mage ability IDs;
- server acknowledgements continue to own cooldown acceptance;
- Babylon cast, projectile, impact and aura effects are connected to combat events;
- icon and 3D effect profiles respect `prefers-reduced-motion`.

## Protocol and persistence

- protocol version 3 and content version 0.3.0 replace the 0.2 identity catalog;
- migration 7 is a rolling-deploy-safe expand step: it adds `gender`, does not rewrite existing race/class rows and allows both legacy and canonical identity IDs;
- shared persistence normalizers map every legacy row to the current five-race/two-class matrix before API, world and locked inventory/economy operations;
- existing `first_echoes`, inventory, economy and character IDs are retained;
- the catalog publishes genders, the six monster definitions and Dawnmere zone metadata.

A cached protocol-2 client must reload. Identity backfill is intentionally deferred: after all 0.2 instances are retired and the rollback window closes, a later migration can backfill in bounded batches; a separate contract release will verify zero legacy rows, narrow constraints and then remove dual-read aliases. Migration 7 itself keeps legacy rows/writes valid, but after 0.3 creates canonical rows a 0.2 application rollback is unsafe; use a 0.3 fix-forward or restore a matched pre-0.3 snapshot together with the 0.2 image.

## Validation

The candidate must pass repository tests, TypeScript, ESLint, asset provenance, the 20-character/6-monster GLB verifier, production builds, Docker build, PostgreSQL integration and multiplayer smoke gates. External backup, restore, deployment and soak evidence is tracked separately in `docs/PRODUCTION_READINESS.md`.

## Deployment

The production Docker image contains both the Vite client and Fastify server. Railway exposes one HTTPS origin for the SPA, `/v1`, `/socket.io`, `/healthz` and `/readyz`; the client resolves API and realtime connections through `window.location.origin`. GitHub Pages and its deploy workflow are no longer part of the release path.

## Known limits

- current classes expose only the basic and one signature ability each;
- no party, friends, trade, guild, auction, dungeon, raid or broad quest campaign yet;
- no AOI/Redis/zone-worker scaling or proven high-concurrency capacity;
- generated GLB assets are technically validated but still need manual AAA-level art passes;
- one start zone and six creatures are a vertical slice, not MMO-scale world content;
- operational readiness must be re-evidenced for the exact 0.3 candidate before any production-ready claim.
